# Руководство по импорту мероприятий

## Быстрый старт

Есть 3 способа добавить мероприятия в приложение:

### Способ 1: CSV файл (самый простой)

1. Создайте файл `events.csv` в Excel или Google Sheets
2. Используйте следующие колонки:

```csv
id,title,description,date,startTime,endTime,zoneId,speaker,capacity
event-1,Открытие форума,Торжественное открытие,2024-03-15,09:00,10:00,zone-1,Иванов И.И.,500
event-2,Мастер-класс,Практический мастер-класс,2024-03-15,10:30,12:00,zone-2,Петрова А.С.,50
event-3,Обед,Обеденный перерыв,2024-03-15,12:00,13:00,zone-3,,200
```

3. Запустите скрипт конвертации:
```bash
node convert_events_csv.js events.csv
```

4. Скопируйте результат в `src/data/mockData.ts`

---

### Способ 2: JavaScript функция (для программистов)

Используйте файл `import_events.js`:

```javascript
const events = [
  createEvent({
    id: 'event-1',
    title: 'Открытие форума',
    description: 'Торжественное открытие форума',
    date: '2024-03-15',
    startTime: '09:00',
    endTime: '10:00',
    zoneId: 'zone-1',
    speaker: 'Иванов И.И.',
    capacity: 500
  }),
  // Добавьте больше...
]
```

Запустите:
```bash
node import_events.js
```

---

### Способ 3: Прямое редактирование (для небольших изменений)

Откройте `src/data/mockData.ts` и добавьте мероприятия напрямую:

```typescript
export const events: Event[] = [
  {
    id: 'event-1',
    title: 'Открытие форума',
    description: 'Торжественное открытие',
    startTime: '2024-03-15T09:00:00',
    endTime: '2024-03-15T10:00:00',
    zoneId: 'zone-1',
    speaker: 'Иванов И.И.',
    capacity: 500
  },
  // ...
]
```

---

## Список доступных зон (zoneId)

Проверьте актуальные ID зон в файле `src/data/mockData.ts` в массиве `zones`.

Типичные зоны:
- `zone-1` - Главный зал
- `zone-2` - Конференц-зал А
- `zone-3` - Конференц-зал Б
- `zone-4` - Выставочная зона
- `zone-5` - Зона регистрации
- `zone-6` - Фуд-корт
- `zone-7` - Лаунж-зона

---

## Формат полей

| Поле | Тип | Обязательно | Пример | Описание |
|------|-----|-------------|--------|----------|
| id | string | Да | `event-1` | Уникальный идентификатор |
| title | string | Да | `Открытие форума` | Название мероприятия |
| description | string | Да | `Торжественное открытие` | Краткое описание |
| date | string | Да | `2024-03-15` | Дата в формате YYYY-MM-DD |
| startTime | string | Да | `09:00` | Время начала HH:MM |
| endTime | string | Да | `10:00` | Время окончания HH:MM |
| zoneId | string | Да | `zone-1` | ID зоны из списка |
| speaker | string | Нет | `Иванов И.И.` | Имя спикера |
| capacity | number | Нет | `500` | Вместимость зала |

## Проверка после импорта

После добавления мероприятий:

1. Запустите приложение: `npm run dev`
2. Откройте панель "Расписание" (иконка календаря)
3. Проверьте, что все мероприятия отображаются
4. Откройте панель "Зоны" и убедитесь, что мероприятия привязаны к правильным зонам
5. Проверьте время начала и окончания

---

## Советы

- **ID мероприятий** должны быть уникальными (event-1, event-2, event-3...)
- **Время** указывайте в 24-часовом формате (09:00, 14:30, 18:00)
- **Дата** в формате YYYY-MM-DD (2024-03-15)
- **zoneId** должен существовать в списке зон
- Если мероприятие без спикера, оставьте поле пустым
- Capacity (вместимость) по умолчанию 100 человек

---

## Поддержка

Если возникли проблемы:
1. Проверьте формат данных
2. Убедитесь, что все обязательные поля заполнены
3. Проверьте, что zoneId существует
4. Проверьте консоль браузера на ошибки (F12)
