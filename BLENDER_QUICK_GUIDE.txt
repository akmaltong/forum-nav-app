ШАГ 1: ОТКРОЙТЕ МОДЕЛЬ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Запустите Blender
2. File → Import → glTF 2.0 (.glb/.gltf)
3. Выберите SM_MFF.glb
4. Import

ШАГ 2: ПРОВЕРЬТЕ ПУСТЫШКИ (EMPTY)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. В правой панели (Outliner) найдите объекты ZONE_*
2. Они должны быть типа "Empty" (иконка креста)
3. Если их нет - создайте вручную (Shift+A → Empty → Plain Axes)

ШАГ 3: ЗАПУСТИТЕ СКРИПТ ЭКСПОРТА
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Перейдите на вкладку "Scripting" (вверху)
2. Нажмите "Open" (кнопка с папкой)
3. Выберите файл: blender_extract_zones.py
4. Нажмите "Run Script" (кнопка ▶️ или Alt+P)

ШАГ 4: ПОЛУЧИТЕ РЕЗУЛЬТАТ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. В нижней части Blender откроется консоль
2. Если консоль не видна: Window → Toggle System Console
3. В консоли появится:
   - Список найденных зон
   - Их координаты
   - Готовый JavaScript код

ШАГ 5: СКОПИРУЙТЕ КОД
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. В консоли найдите блок:
   "JavaScript format for mockData.ts:"
2. Скопируйте весь код от "export const zones" до "]"
3. ИЛИ откройте файл zone_coordinates.json в папке с моделью

ШАГ 6: ПРИМЕНИТЕ В ПРИЛОЖЕНИИ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Откройте: forum-nav-app/src/data/mockData.ts
2. Найдите: export const zones: Zone[] = [
3. Замените весь массив на скопированный код
4. Сохраните файл (Ctrl+S)
5. Обновите браузер (F5)

Blender:               Приложение:
   Z ↑                    Y ↑
   |  Y                   |  Z
   | /                    | /
   |/                     |/
   +----→ X               +----→ X

Скрипт автоматически конвертирует координаты!

╔════════════════════════════════════════════════════════════════╗
║                  ПРОБЛЕМЫ И РЕШЕНИЯ                            ║
╚════════════════════════════════════════════════════════════════╝

❌ Проблема: "No ZONE_ objects found"
✅ Решение: Убедитесь что пустышки названы ZONE_НазваниеЗоны

❌ Проблема: Консоль не открывается
✅ Решение: Window → Toggle System Console

❌ Проблема: Скрипт не запускается
✅ Решение: Убедитесь что выбрана вкладка "Scripting"

❌ Проблема: Зоны на неправильной высоте
✅ Решение: Переместите Empty на уровень пола (Z ≈ 0)

❌ Проблема: Зоны в одной линии
✅ Решение: Разместите Empty в центрах залов вручную

╔════════════════════════════════════════════════════════════════╗
║             СОЗДАНИЕ ПУСТЫШЕК ВРУЧНУЮ                          ║
╚════════════════════════════════════════════════════════════════╝

1. В 3D окне: Shift + A
2. Empty → Plain Axes
3. Переместите в центр зала:
   - Нажмите G (Grab) для перемещения
   - Нажмите X/Y/Z для движения по оси
   - Введите число или двигайте мышью
4. Переименуйте (F2): ZONE_НазваниеЗоны
5. Повторите для каждой зоны

╔════════════════════════════════════════════════════════════════╗
║                    ГОРЯЧИЕ КЛАВИШИ                             ║
╚════════════════════════════════════════════════════════════════╝

Shift + A       Добавить объект
G               Переместить (Grab)
R               Поворот (Rotate)
S               Масштаб (Scale)
F2              Переименовать
N               Боковая панель
Numpad 7        Вид сверху
Numpad 1        Вид спереди
Numpad 3        Вид сбоку
Alt + P         Запустить скрипт
